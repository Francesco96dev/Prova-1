Attributi generici in HTML

Questi sono alcuni degli attributi generici più comuni in HTML.

  • id: Fornisce un identificatore univoco per l'elemento.
  • class: Assegna una o più classi a un elemento per applicare stili CSS.
  • style: Applica stili CSS inline all'elemento.
  • title: Fornisce un titolo descrittivo per l'elemento.
  • lang: Specifica la lingua del contenuto dell'elemento.
  • dir: Specifica la direzione del testo (left-to-right o right-to-left).
  • tabindex: Definisce l'ordine di tabulazione per gli elementi interattivi.
  • data-*: Consente di aggiungere attributi personalizzati per dati specifici.
Questi attributi sono utili per migliorare l'accessibilità, la semantica e lo stile dei documenti HTML.

attributi per eventi in HTML

Gli attributi per eventi in HTML consentono di gestire le interazioni dell'utente con gli elementi della pagina. Ecco alcuni degli attributi più comuni

  • onclick: Si attiva quando l'elemento viene cliccato.
  • onmouseover: Si attiva quando il mouse passa sopra l'elemento.
  • onmouseout: Si attiva quando il mouse esce dall'elemento.
  • onkeydown: Si attiva quando un tasto viene premuto.
  • onkeyup: Si attiva quando un tasto viene rilasciato.
  • onsubmit: Si attiva quando un modulo viene inviato.
  • onfocus: Si attiva quando un elemento riceve il focus.
Questi attributi possono essere utilizzati per aggiungere interattività e dinamismo alle pagine web, consentendo agli sviluppatori di rispondere agli eventi degli utenti in modo efficace.

Attributi meno conosciuti in HTML

Oltre agli attributi più comuni, HTML offre anche alcuni attributi meno conosciuti ma utili per specifiche esigenze. Ecco alcuni esempi:

  • hidden: nasconde completamente un elemento
  • translate: L’attributo `translate` in HTML serve a indicare ai servizi di traduzione automatica (come Google Translate) se il contenuto di un elemento deve essere tradotto o meno. - `translate="yes"` (o `translate="true"`): il contenuto può essere tradotto. - `translate="no"` (o `translate="false"`): il contenuto **non** deve essere tradotto. Ad esempio, se si ha un marchio o un nome proprio che non si desidera tradurre, si può utilizzare `translate="no"` per evitare che venga modificato dai servizi di traduzione automatica.
  • aria-label: fornisce un'etichetta per l'accessibilità agli screen reader
  • aria-hidden: nasconde un elemento agli screen reader
